Appcrawler自动遍历思路

信息的获取:

  • 把当前app的界面dump为xml结构
    获取待遍历元素
  • 遍历范围 selectedList
  • 过滤黑名单小控件不可见控件 blackList
  • 重排控件顺序 firstList lastList
  • 跳过已点击 + 跳过限制点击的控件tagLimit
  • 根据匹配的规则执行action

循环上面的步骤

优点

1.可以定义遍历的路径
2.可实现自动输入, 自动滑动
3.支持黑白名单的遍历
4.支持优先遍历tab元素
（如优先遍历卡牛首页”我的”里面所有元素，再遍历”账户”下面所有元素）
5.支持遍历路径的深度
6.支持编写测试用例
7.良好的结果分析:
a.点击前后的截图对比
b.新老版本的ui diff
c.app结构思维导图

缺点

1.遍历元素较慢
2.无法遍历覆盖所有的场景

Appcrawler存在的bug

1.用appcrawler遍历apk，运行久了会导致内存不够
例如：”Exception in thread “main” java.lang.OutOfMemoryError:java heap space”

2.目前设置的黑名单里面的元素仍然会去点击

3.在遍历的时候点击元素跳转到其他apk，会重启app并且会清除app所有的数据以及登录状态
例如（会清除卡牛导入的网银卡片和登录的状态）

（以上所有问题都已向开源作者反馈，待后续优化解决）

环境

配置java 环境变量：

JAVA_HOME 变量值为JDK的安装目录,笔者为C:\Program Files\Java\jdk1.8.0_91
PATH PATH变量值后追加;%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin;
CLASSPATH 变量值为%JAVA_HOME%\lib;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar;

配置android-sdk环境变量:

ANDROID_HOME 变量值为android-sdk的解压目录，笔者为D:\Android\android-sdk-windows
PATH PATH变量值的最后追加 ;%ANDROID_HOME%\tools;%ANDROID_HOME%\platform-tools;%ANDROID_HOME%\build-tools;

对应的支持的版本：
appcrawler 2.1.1 支持appium1.7.1 java8

appcrawler 2.1 支持appium 1.6.4 java6

appium不同版本差别很大，兼容性支持很弱

备注：（android-sdk、java的环境变量配置，最好是相对路径。如果是绝对路径，运行appcrawler会提示”android-home can’t find”）

使用

以下3种方法运行之前都要开启appium
1.裸跑（也就是不带配置文件跑，用默认的配置跑，看环境是否配置正确）：
a.打开被测的apk
b.java -jar appcrawler-2.1.2.jar -u http://0.0.0.0:4723/wd/hub -o result/

2.裸跑2（配置capablility，告诉appcrawler需要运行的app以及它的activity）
java -jar appcrawler-2.1.3.jar –capability “appPackage=’com.mymoney.sms’,appActivity=’/.ui.main.MainActivity” -u http://127.0.0.1:4730/wd/hub -o result/

注意：仅在windows环境下需要加双引号
“appPackage=’com.mymoney.sms’,appActivity=’/.ui.main.MainActivity”

3.定制化自动遍历
java -jar appcrawler-2.1.3 -c conf/mymoney_sms.yml
(建议将编写的配置文件mymoney_sms.yml放在conf目录下)

配置文件的参数经验

部分配置：
capability的配置：
androidCapability:
deviceName: “demo”
appPackage: “com.mymoney.sms”
appActivity: “.view.WelcomeActivityAlias”
app: “”
appium: “http://127.0.0.1:4723/wd/hub"
fullReset: false
((iOS) 删除整个模拟器目录。(Android) 通过卸载而不是清空数据—来重置应用状态。在 Android 上，这也会在会话结束后自动清除被测应用。默认值 false）
noReset: true
(不要在会话前重置应用状态，默认值false。如果想要保持已登录状态，设置为true)
automationName: uiautomator2 （除uiautomator2外还支持macaca、appium）

dontStopAppOnReset: true
（在使用 adb 启动应用时不要停止被测应用的进程。默认值： false）

testcase配置：
testcase: #支持自动化测试，given，when,then可以默认置空，只需要xpath和action进行配置

以上action目前仅支持:

  • “”只是截图记录
  • back 后退
  • backApp 回退到当前的app 默认等价于back行为可定制
  • monkey 随机事件
  • xxx() 执行代码
    • Thread.sleep(3000)
    • driver.swipe(0.9, 0.5, 0.1, 0.5)
  • click
  • longTap

xpathAttributes: (xpath目前支持)

selectedList:

(selectedList,以上就是点击clickable=True的元素)

urlBackList:

（urlBackList是黑名单的意思，以上就是把点击可能跳转到其他apk去的元素（如卡牛的”支付宝付款”和”微信付款”）或者不想去遍历的元素（如卡牛的”投资”和”理财”的Tab）屏蔽。不过目前存在bug，被屏蔽的元素还是会去点击。）

FAQ

1.Exception in thread “main” java.lang.ExceptionInInitializerError
Caused by:java.lang.NullPointerException
原因：这个是由于装了orcal的版本的jdk与java的jdk环境不一致
解决办法:将jdk的配置置于前面，并删除oracle的classpath的配置

2.[AndroidDriver] Error: Unable to find an active device or emulator with OS 6.1. The following are available: 1c654712(4.3)
带配置文件跑的时候出现如下提示：(WARNING: The server did not provide any stacktrace information)
原因:编写的配置文件有问题
解决方法：
appium运行自动化脚本用java -jar appcrawler-2.1.2.jar –demo生成默认配置文件，并在notepad编辑进行保存。

3.运行完testcase，提示NullPointerException: at com.testerhome.appcrawler.Crawler.getElementByElementActionsCrawler.scala:1116
原因:这个是我们在获取元素时，配置文件triggerActions列表里面是空的
解决办法:我们需要在设置相应的返回元素信息。

4.[JSONWP Proxy] Proxying [GET /status] to [GET http://localhost:8200/wd/hub/status] with no body
原因：capbility的automation设置的有问题
解决办法:检查capbility配置的automation(将uiautomotor2改为appium)
